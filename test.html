wav to mp3; mp3 to wav

#!/bin/sh
mp32wav() {
	[[ $# -eq 0 ]] && ( for f in *.mp3 ; do mpg321 -w ${f%.mp3}.wav $f &>/dev/null; done ) || ([[ -f $1 ]] && mpg321 -w ${1%.mp3}.wav $1 || echo "Invalid parameter!")
}
wav2mp3() {
	[[ $# -eq 0 ]] && ( for f in *.wav ; do lame -h -b 192 $f ${f%.wav}.mp3 &>/dev/null; done ) || ([[ -f $1 ]] && lame -h -b 192 ${1%.wav}.mp3 || echo "Invalid parameter!")
}

linux and ssd

source: http://superuser.com/questions/228657/which-linux-filesystem-works-best-with-ssd

By default, many distributions including Ubuntu use the relatime flag for updating file metadata when files are accessed, but you're unlikely to care about last access times. Additionally, Linux supports TRIM with Ext4. TRIM is important for maintaining the performance of an SSD over time as files are added, deleted and changed and lets the SSD know which blocks can be safely cleared. No distributions currently enable it by default, but it's simple to do by adding the discard flag to any mounted SSDs. To make all these changes, open up a terminal and run: 

sudo nano -w /etc/fstab
Then for all SSD devices in your system remove relatime if present and add noatime,discard so it looks something like this: 
/dev/sda / ext4 noatime,discard,errors=remount-ro 0 1

Create a video from an image and a sound file

avconv -loop 1 -i image.png -i sound.mp3 -c:v libx264 -c:a aac -strict experimental -b:a 320k -shortest out.mp4
Without re-encoding the sound file: 
avconv -loop 1 -i image.png -i audio.m4a -c:v libx264 -c:a copy -shortest out.mp4
Without sound (-t in seconds): 
avconv -loop 1 -i image.png -c:v libx264 -t 30 -pix_fmt yuv420p out.mp4
